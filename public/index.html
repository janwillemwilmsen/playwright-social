og:image aspect ratio
:
The minimum allowed image dimension is 200 x 200 pixels. The size of the image file must not exceed 8 MB. Use images that are at least 1200 x 630 pixels for the best display on high resolution devices. At the minimum, you should use images that are 600 x 315 pixels to display link page posts with larger images. If your image is smaller than 600 x 315 px, it will still display in the link page post, but the size will be much smaller. Try to keep your images as close to 1.91:1 aspect ratio for images. So they are the same across desktop and mobile Feed. Use og:image:width and og:image:height, it will specify the image dimensions to the facebook crawler so that it can render the image immediately without having to asynchronously download and process it.

    LinkedIn

    LinkedIn uses oEmbeds and/or Open Graph Protocol (OGP) and other data from the page to show shared pages in the feed. The Linkedin Post Inspector does not always show the right preview. Have tested many websites in it, and checked the results in the LinkedIn app. So we cannot be sure about what will shared when the oEmbed or Open Graph data isn't available. The developer documentation is lacking information about the lenght of descriptions and and wether or not they are shown, or what the fallback value will be. The documentation only states that you should use :

    • 'og:title
    • 'og:image' --> Max file size: 5 MB // Minimum image dimensions: 1200 (w) x 627 (h) pixels // Recommended ratio: 1.91:1
    • 'og:description'
    • 'og:url'

    If the image isn't visable when shared that might be because of server security settings, which prohibit embedding data and resources on other platforms. Also when pages with a small og:images is shared, the image is on the left of the post(small) instead of the top of the post(large).

      WhatsApp

      WhatsApp uses open graph and other data from the page to show shared pages in the feed. If og:image is squared, Whatsapp will show larger square image. og:image also needs to be below 300kb (but I cannot test image filesize yet, am only using height and width....). Other unoffical documentation states: the image needed to be under 300kb in size and 256x256px but not larger than 400x400px.

      Note : it is possible to add multiple og:images to a website or page. Besides the second 'og:image' you need to add 'og:image:type', 'og:image:width', and 'og:image:height' attributes for both og:images.
